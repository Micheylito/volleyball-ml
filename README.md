# Volleyball ML Signals

Стартовый проект для вашей волейбольной базы:

- подключение к базе на Ubuntu;
- выгрузка матчей в датасет;
- обучение модели;
- генерация сигналов;
- отправка сигналов в Telegram;
- работа через GitHub как центральную точку синхронизации.

## 1. Как мы будем работать

Простая схема:

1. Код хранится в GitHub.
2. Ubuntu-сервер хранит базу и, при желании, скрипт-сборщик.
3. Codex работает с репозиторием и помогает писать код.
4. ML-скрипт забирает данные из базы, обучается и сохраняет модель.
5. Скрипт сигналов анализирует новые матчи и отправляет сообщения в Telegram.

## 2. Первый путь от нуля до результата

### Шаг 1. Создайте пустой репозиторий на GitHub

Создайте новый репозиторий, например `volleyball-ml-signals`.

Рекомендуемые настройки:

- `Public` или `Private` на ваш выбор;
- без `README`, если будете загружать этот проект из локальной папки;
- без `.gitignore`, так как он уже есть в проекте.

### Шаг 2. Привяжите локальную папку к GitHub

В PowerShell из корня проекта:

```powershell
git init
git add .
git commit -m "Initial project scaffold"
git branch -M main
git remote add origin https://github.com/YOUR_GITHUB_LOGIN/YOUR_REPO_NAME.git
git push -u origin main
```

Если Git попросит логин, обычно удобнее авторизоваться через Git Credential Manager или GitHub Desktop.

### Шаг 3. Подготовьте Ubuntu

На сервере должны быть:

- Python 3.11+;
- доступ к вашей волейбольной БД;
- `git`;
- желательно `venv`.

Пример команд на Ubuntu:

```bash
sudo apt update
sudo apt install -y git python3 python3-venv python3-pip
git clone https://github.com/YOUR_GITHUB_LOGIN/YOUR_REPO_NAME.git
cd YOUR_REPO_NAME
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
```

После этого заполните `.env` реальными значениями.

### Шаг 4. Подключите базу

В `.env` укажите строку подключения:

```env
DB_URL=postgresql+psycopg2://user:password@server_ip:5432/database_name
```

Если база не PostgreSQL, позже поменяем драйвер и код подключения.

### Шаг 5. Проверьте экспорт матчей

```bash
python -m src.train
```

Скрипт попробуется:

- выгрузить данные из базы;
- построить признаки;
- обучить простую стартовую модель;
- сохранить ее в `models/`.

### Шаг 6. Запустите генерацию сигнала

```bash
python -m src.signals
```

Если вероятность выше порога, скрипт отправит сообщение в Telegram.

## 3. Как подключить Codex через GitHub к Ubuntu

Прямое удобное соединение обычно строят так:

1. Ubuntu-сервер и ваш локальный компьютер работают с одним GitHub-репозиторием.
2. Codex редактирует код в репозитории.
3. Вы пушите изменения в GitHub.
4. На Ubuntu делаете `git pull`.
5. Скрипты на сервере запускаются уже на свежем коде.

Это самый надежный путь для старта, особенно если GitHub у вас впервые.

Если позже захотите, можно автоматизировать деплой:

- через GitHub Actions;
- через webhook;
- через `systemd` и `cron` на Ubuntu.

Но на первом этапе лучше не усложнять.

## 4. Что нужно от вашей текущей базы, чтобы сделать модель нормально

Минимально полезные поля по матчу:

- дата матча;
- лига;
- домашняя команда;
- гостевая команда;
- итоговый победитель;
- счет по сетам;
- коэффициенты букмекера до матча;
- форма команд за последние матчи;
- статистика очков, если есть;
- статус матча: доигран/отменен.

Чем лучше и чище историческая база, тем лучше сигналы.

5. Проверить первое обучение.
6. Подключить Telegram.
7. Настроить регулярный запуск на сервере.
